# ТОО — Учёт смен и расходов

Система учёта смен, зарплат, техники, доходов и расходов для ТОО (строительная/крановая компания).

## Возможности

- Табель смен по каждому крану (дневные, ночные, сверхурочные)
- Автоматический расчёт зарплат операторов с учётом замен
- Учёт доходов с НДС 16%
- Учёт расходов по категориям
- Отчёт о прибылях и убытках
- Период расчётов: 21 число — 20 число следующего месяца
- Повторяющиеся статьи с копированием на новый период
- Экспорт / импорт данных в JSON
- Работает офлайн (PWA)

## Публикация на GitHub Pages

### Шаг 1 — Подготовьте иконки

1. Откройте файл **`create-icons.html`** в браузере (Chrome или Edge)
2. Нажмите **«Скачать все иконки»** — скачается 4 PNG файла
3. Поместите их в эту же папку рядом с `smeny.html`:
   - `icon-192.png`
   - `icon-512.png`
   - `icon-maskable.png`
   - `apple-touch-icon.png`

### Шаг 2 — Создайте репозиторий на GitHub

1. Зайдите на [github.com](https://github.com) и войдите в аккаунт
2. Нажмите **«New repository»** (зелёная кнопка)
3. Задайте имя, например: `too-uchet`
4. Выберите **Public** (иначе GitHub Pages будет платным)
5. **Не ставьте галочки** на README, .gitignore, licence
6. Нажмите **«Create repository»**

### Шаг 3 — Загрузите файлы

**Способ А — через браузер (проще):**

1. На странице репозитория нажмите **«uploading an existing file»** (или «Add file → Upload files»)
2. Перетащите все файлы из папки в окно браузера:
   - `smeny.html` → переименуйте в `index.html` перед загрузкой
   - `manifest.json`
   - `sw.js`
   - `icon.svg`
   - `icon-192.png`
   - `icon-512.png`
   - `icon-maskable.png`
   - `apple-touch-icon.png`
   - `.nojekyll`
3. Нажмите **«Commit changes»**

> **Важно:** файл `smeny.html` должен называться `index.html` на GitHub, иначе GitHub Pages не откроет его по умолчанию.

**Способ Б — через Git (если установлен):**

```bash
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/too-uchet.git
git push -u origin main
```

### Шаг 4 — Включите GitHub Pages

1. Зайдите в **Settings** вашего репозитория
2. Прокрутите до раздела **Pages** (левое меню)
3. В разделе **«Build and deployment»** выберите:
   - Source: **Deploy from a branch**
   - Branch: **main** / **(root)**
4. Нажмите **Save**
5. Подождите 1–2 минуты

Ваш сайт будет доступен по адресу:
```
https://ВАШ_ЛОГИН.github.io/too-uchet/
```

## Установка на телефон (PWA)

### iPhone / iPad

1. Откройте сайт в **Safari** (только Safari — Chrome на iOS не поддерживает)
2. Нажмите кнопку **«Поделиться»** (квадрат со стрелкой вверх)
3. Выберите **«На экран «Домой»»**
4. Нажмите **«Добавить»**

Приложение появится на главном экране и будет работать как нативное.

### Android

1. Откройте сайт в **Chrome**
2. Нажмите **⋮** (три точки) → **«Добавить на главный экран»** или **«Установить приложение»**
3. Подтвердите установку

## Обновление данных

Данные хранятся в **localStorage браузера** на вашем устройстве.

- Данные на телефоне и компьютере — **разные** (браузеры не синхронизируются)
- Для переноса данных используйте **«Экспорт JSON»** и **«Импорт JSON»** в настройках

## Структура файлов

```
index.html           ← основное приложение (smeny.html → переименовать)
manifest.json        ← PWA манифест
sw.js                ← Service Worker (офлайн-режим)
icon.svg             ← векторная иконка
icon-192.png         ← иконка Android 192×192
icon-512.png         ← иконка для splash-экрана
icon-maskable.png    ← адаптивная иконка Android
apple-touch-icon.png ← иконка для iPhone/iPad
.nojekyll            ← отключает Jekyll на GitHub Pages
create-icons.html    ← генератор PNG иконок (не нужен на сайте)
```

## Обновление приложения

При изменении кода увеличьте версию кэша в `sw.js`:

```js
const CACHE_VERSION = 'v3'; // было v2 → стало v3
```

Это заставит все устройства обновить кэш при следующем открытии.
